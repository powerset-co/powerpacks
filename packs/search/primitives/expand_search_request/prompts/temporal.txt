## 1. POSITION DATE RANGE

Extract the time period when someone held a position at a company. This answers: "when did they work there?"

⚠️ CRITICAL: ONLY set position_start_year / position_end_year when the query contains EXPLICIT YEARS or date references.
Do NOT infer dates from words like "currently", "now", "recently" — those are handled by is_current (see below).

Explicit ranges:
- "worked at Google 2018-2020" → position_start_year: 2018, position_end_year: 2020
- "at Stripe between 2019 and 2022" → position_start_year: 2019, position_end_year: 2022
- "at Thumbtack from 2012 to 2015" → position_start_year: 2012, position_end_year: 2015
- "at Meta 2012-2015" → position_start_year: 2012, position_end_year: 2015

Open-ended ranges (with explicit years):
- "worked at Google after 2020" → position_start_year: 2020, position_end_year: null
- "at Stripe before 2018" → position_start_year: null, position_end_year: 2018
- "since 2019" → position_start_year: 2019, position_end_year: null
- "post-2020" → position_start_year: 2020, position_end_year: null
- "pre-2015" → position_start_year: null, position_end_year: 2015

Approximate/fuzzy (with year references):
- "around 2020" → position_start_year: 2019, position_end_year: 2021
- "circa 2015" → position_start_year: 2014, position_end_year: 2016
- "in the early 2010s" → position_start_year: 2010, position_end_year: 2013
- "in the mid 2010s" → position_start_year: 2013, position_end_year: 2016
- "in the late 2010s" → position_start_year: 2017, position_end_year: 2019
- "during COVID" → position_start_year: 2020, position_end_year: 2021
- "in YYYY" → position_start_year: YYYY, position_end_year: YYYY

Single year in context of a company:
- "at Google in 2020" → position_start_year: 2020, position_end_year: 2020

⚠️ Do NOT set position dates for these — use is_current instead:
- "currently working at" → is_current: true, NO position dates
- "now at Google" → is_current: true, NO position dates
- "recently joined" → is_current: true, NO position dates
- "previously at Google" → is_current: false, NO position dates
- "formerly at" → is_current: false, NO position dates
- "a few years ago" → is_current: false, NO position dates

⚠️ Do NOT confuse with graduation years or age.

## 2. CURRENT/PAST POSITION (is_current)

Determines whether the user is asking about someone's CURRENT role, a PAST role, or doesn't care.

Set is_current: true when:
- "currently working at/as" → true
- "now at" → true
- "people who are currently" → true
- "presently" → true

Set is_current: false when:
- "previously at" / "formerly at" → false
- "used to work at" → false
- "past employees of" → false
- "ex-Google" / "former Stripe" → false
- "who left" / "who departed" → false
- "alumni of [company]" → false
- "a few years ago" → false

Set is_current: null (default) when:
- No mention of current/past → null
- "engineers at Google" (ambiguous) → null
- "people who worked at Google" (could be current or past) → null
- "software engineers in SF" → null

⚠️ CRITICAL: Default is null. Only set true/false when EXPLICITLY mentioned.

Relative (compute from current year 2026):

- "graduated in the last 5 years" → graduation_year_min: 2021, graduation_year_max: 2026
- "graduated in the last 3 years" → graduation_year_min: 2023, graduation_year_max: 2026

⚠️ CRITICAL: Do NOT confuse graduation years with position dates.
- "Stanford grads 2018-2022" → graduation years (about when they graduated)
- "worked at Google 2018-2020" → position dates (about when they worked there)
